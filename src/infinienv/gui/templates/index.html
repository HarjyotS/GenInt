Foundry online
A model writes and runs its own engine code in an isolated per-run copy, plays through it, and repairs against an independent outer check + faithfulness audit. The scene it declares is run through the real deterministic validator (bounds/ids enforced, the rest recorded). Needs an OpenAI key + pip install -e ".[openai]".
Which agent writes and runs the engine code. Interchangeable -- same workspace, artifacts, and checks. Claude needs CL_KEY/ANTHROPIC_API_KEY, the claude CLI on PATH, and pip install infinienv[claude].
Which model the agent runs as. Options change with the runtime above. Runs are long and iterative, so a cheaper/faster tier is a fine default; pick a more capable one for the hardest prompts.
Sprites the agent resolves inside its workspace.
A pixel-only vision policy plays a generated world seeing only rendered frames, and a code-defined reward (is_goal_complete) decides success — never the pixels. A naive VLM-on-pixels verdict is recorded beside it, so a disagreement shows why the code signal is the trustworthy one. Needs a vision key.
Curated example worlds, or any world you generated (run, including sandbox). A sandbox world is played faithfully — the vision policy drives its real game (its own side-view frames + physics + win) inside the sandbox. A deterministic/example scene is played through the top-down engine. Either way, success is judged by code.
Sprites drawn into the frames the policy sees.
Play a world yourself with the keyboard. You drive the deterministic engine (InfiniEnv) directly — arrow keys / WASD to move, Space or E to interact — and reaching the goal is judged by code (is_goal_complete), never the pixels. Works for every scene; a sandbox world plays as its declared top-down layout.
